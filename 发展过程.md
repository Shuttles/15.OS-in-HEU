# 无操作系统的计算机系统

## 特征

即人工操作方式

1. 1946~50年代初(电子管)，集中计算(计算中心)，计算资源昂贵
2. 工作方式
   + 用户：用户既是程序员，又是操作员；用户是计算机专业人员。
   + 编程语言：机器语言
   + 输入输出：纸带或卡片
3. 计算机的工作特点
   + 用户独占全机：不出现资源被其他用户占用，资源利用率低
   + CPU等待用户：计算前，手工装入纸带或卡片；计算完成后，手工卸取纸带或卡片；CPU利用率低。



***<u>严重降低了计算机资源的利用率--人机矛盾！</u>***

硬件不断发展，CPU速度的提高、系统规模扩大，人机矛盾严重，如何解决？



脱机输入输出(off-Line I/O)方式

1. 事先将用户程序或数据的纸带或卡片装入纸带输入机，在外围机的控制下输入到磁带上，CPU从磁带上读入数据；输出过程正好相反。

2. 特点

   + 减少了CPU的空闲时间
   + 提高I/O速度

## 主要矛盾

1. 计算机处理能力的提高，手工操作的低效率(造成浪费)
2. 用户独占全机的所有资源



## 提高效率的途径

专门的操作员，批处理





# 单道批处理系统

1. 50~60年代(晶体管) ：利用磁带把若干个作业分类编成作业执行序列，每批作业由一个专门的监督程序(Monitor)自动依次处理。可使用汇编语言开发。

2. 批处理中的作业的组成

   + 用户程序
   + 数据
   + 作业说明书(作业控制语言)

3. 批

   供一次加载的磁带或磁盘，通常由若干个作业组装成，在处理中使用一组相同的系统软件(系统带)

## 特征

1. 自动性

   在顺利的情况下，磁带上的一批作业能自动地逐个地依次运行，而无需人工干预。

2. 顺序性

   磁带上的各道作业是顺序地进入内存，完成顺序与进入内存顺序相同

3. 单道性

   内存中只有一道程序运行

4. CPU和I/O设备使用忙闲不均(取决于当前作业的特性)

   + 对计算为主的作业，I/O空闲
   + 对I/O为主的作业，CPU空闲

   

# 多道批处理系统

即Multiprogrammed Batch System

在单道批处理系统中，CPU、I/O忙闲不均、内存利用率低，如何解决？

1. 60年代中~70年代中(集成电路)，利用多道批处理系统提高资源的利用率

2. 简单来说

   在当前运行的作业需做I/O处理时，CPU转而执行另一个作业

3. 示意图

   ![img](https://wx4.sinaimg.cn/mw690/005LasY6gy1gcq0offqddj30m20gc3zu.jpg)



## 特征

1. 调度性

   作业提交给系统需经过两次调度

   + 作业调度，从后备队列进入内存；

   + 进程调度，分配CPU运行。

2. 无序性

   多个作业完成的先后顺序与他们进入内存的顺序之间无对应关系

3. 多道性

   + ***<u>内存中同时存放几个作业</u>***，从而有效地提高了资源利用率
   + 作业可以***<u>并发执行</u>***，这提高了系统的吞吐量



## 优缺点

优点

1. 资源利用率高：CPU、I/O和内存利用率高
2. 作业吞吐量大：单位时间内完成的工作量大



缺点

1. 用户交互性差

   整个作业完成后或中间出错时，才与用户交互，不利于调试和修改

2. 作业平均周转时间长

   短作业的周转时间显著增长
